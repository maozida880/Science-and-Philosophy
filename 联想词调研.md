根据补充信息，需要调整原提纲中的几个核心判断：

1. **不能再泛泛讨论“排序模型是否个性化”**，因为已经明确当前 LR 公式已存在，且没有 User Profile 特征，需要重点分析 **Item侧排序逻辑是否合理、领域目标是否一致、模型是否能承接联想词流量**。
2. **联想词（Suggestion）需要从一个普通 NLP 模块提升为核心业务增长入口**，因为它贡献绝大部分订单和金额，应该作为专项调研对象。
3. **调研重点应从“搜索系统整体能力”倾斜到“Query进入搜索前的决策链路”**，即：

   > 用户输入什么 → 联想展示什么 → 用户点击什么 → 最终成交什么。

下面是调整后的关键章节（可替换原提纲对应部分）。

# 二、联想词体系专项调研（Suggestion Optimization）

## 调研目标

当前搜索产能高度集中于联想词区域。

根据5月业务数据：

* 联想搜索区域订单量超过11.5万；
* 成交金额超过25.3亿元；

明显高于历史搜索、投资理财等入口。

因此，联想词并非简单的 Query 辅助功能，而是当前搜索业务的**核心流量分发入口**。

本阶段重点需要回答：

> 当前联想词是在帮助用户表达需求，还是已经成为业务产品推荐入口？

---

# 1. 联想词生成机制分析

重点调研当前 Suggestion 的生成链路：

## （1）候选词来源

候选词主要来自：

* 用户搜索日志？

| 模型生成词 |    |      |

核心判断：

> 当前联想词优化空间主要来自“扩大召回”，还是“提升候选排序能力”？

# 2. 联想词排序逻辑调研

当前联想词承担主要成交贡献，需要重点拆解排序模型。

重点确认：

目前排序是否基于：

* 搜索频次
* 点击率 CTR
* 转化率 CVR
* GMV
* 产品利润
* 运营权重
* 人工Boost

是否存在类似：

```
Suggestion Score =
a × Search Frequency
+
b × CTR
+
c × CVR
+
d × GMV
+
e × Business Weight
```

的计算逻辑。

重点分析：

### 高流量词是否一定优先？

例如：

“信用卡”

可能展示：

* 信用卡申请
* 信用卡还款
* 信用卡优惠

哪个应该排第一？

排序目标到底是：

* 最大点击？
* 最大成交？
* 最大用户满意度？

核心问题：

> 联想词目前优化目标是否和搜索商业目标一致？

---

# 3. 联想词个性化能力评估

目前系统没有用户个性化处理。

需要重点评估：

对于金融搜索场景，没有用户上下文是否存在明显损失。

例如：

用户A：

* 已持有基金产品

搜索：

“基金”

更合理：

* 基金收益查询
* 基金赎回
* 基金加仓

用户B：

* 未购买基金

更合理：

* 基金开户
* 热门基金推荐

当前系统如果统一返回：

意味着：

> 搜索入口仍然是“公共排行榜”，而不是用户经营入口。

重点调研：

未来是否引入：

* 用户资产标签
* 产品持有情况
* 生命周期阶段
* 风险等级
* 最近行为

---

# 4. 联想词展示策略分析

重点关注：

## 首屏资源分配

当前：

* 展示几个联想词？
* 是否固定排序？
* 是否存在动态调整？

需要分析：

Top1、Top3、Top5：

* 点击贡献
* 成交贡献

判断：

> 当前收益主要集中在头部位置，还是长尾联想词贡献明显？

# 5. 联想词Bad Case分析

这类词是联想优化的重要机会。

# 五、排序模型与业务规则机制调研（调整版）

## 1. 当前LR排序体系分析

目前已明确：

* 各领域存在独立LR排序公式；
* 当前无用户个性化特征。

因此重点不再是“是否有模型”，而是：

### 模型目标是否合理？

需要调研：

不同领域LR优化目标：

例如：

金融产品：

是否：

```
Score =
w1 × CTR
+
w2 × CVR
+
w3 × GMV
```

商城：

是否：

```
Score =
w1 × 点击
+
w2 × 销售额
+
w3 × 热度
```

服务：

是否：

```
Score =
w1 × 相关性
+
w2 × 使用频率
```

重点关注：

不同业务域：

* 是否统一目标？
* 是否存在业务指标冲突？

---

## 2. LR模型与人工策略融合

当前存在：

* Boost提权
* 精选排序
* 销售状态排序

需要明确：

最终排序链路：

例如：

```
Query理解

↓

召回

↓

LR排序

↓

业务规则调整

↓

人工Boost

↓

最终结果
```

还是：

```
人工规则
覆盖
模型结果
```

核心判断：

> 当前搜索排序到底是模型驱动，还是策略驱动？
